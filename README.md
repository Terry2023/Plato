🏛️ Plato: Asynchronous Multi-Agent Concurrency & Urgency Gate Framework
An event-driven, microservice-based agent environment designed to facilitate organic, non-linear multi-agent interactions. Operating entirely on local silicon via WSL/Ubuntu (AMD 7950X 16-Core / RTX 4070 / 64GB RAM), this repository provides the structural behavioral playground where containerized agent profiles actively negotiate, debate, and compete for execution locks based on dynamic linguistic urgency metrics.

🧬 Architectural Overview
Standard multi-agent frameworks rely on strict, deterministic loops (e.g., round-robin or centralized sequential chains) to pass execution control from one agent to the next. This creates artificial conversation pacing, strips out defensive cognitive friction, and forces agents to default to homogenous, passive agreement.

Plato solves this by decoupling the agents into autonomous, isolated microservices managed by a central asynchronous gatekeeper (gateway.py). Instead of speaking when programmatically forced, agents choose when to interrupt or respond based on how their localized psychological traits react to the context stream.
